# Certificates

Mox uses a Let's Encrypt SAN certificate issued by lego DNS-01 via Cloudflare, managed by the `mox-lego-cert` systemd service/timer.

## Architecture

Gerbil/Traefik owns public ports 80 and 443. Mox cannot use normal ACME TLS-ALPN (needs port 443). Instead, DNS-01 ACME with Cloudflare API is used, independent of Traefik's own ACME.

## Certificate Issuance

The NixOS module at `modules/mox/cert-renewal.nix` defines:

- `mox-lego-cert.service` — oneshot that runs lego to issue/renew
- `mox-lego-cert.timer` — daily at randomized time (+3h), persistent

Lego command (generated by Nix):

```text
lego run --server letsencrypt --ipv4only --ari-disable --accept-tos \
  --email noor@latif.se --dns cloudflare \
  --env-file /var/lib/acme/fullstacked-cloudflare.env \
  --path /var/lib/mox-lego --cert.name fullstacked-mail --key-type EC256 \
  -d mail.fullstacked.se -d mta-sts.fullstacked.se -d autoconfig.fullstacked.se
```

`--ipv4only` is required on this VPS: production LE requests fail without it (staging works).
`--ari-disable` works around lego v5 ARI compatibility issue. Can be removed once lego stabilizes.

Lego v5.2.2 is provided via the flake overlay (NixOS `security.acme` emits lego 4 flags and is not used).

## Dual Copy

After issuance, the service copies cert+key to two locations:

1. **Mox TLS** — `/var/lib/mox/config/tls/`

   ```text
   fullstacked-mail-chain.pem  (root:mox 0640)
   fullstacked-mail-key.pem    (root:mox 0640)
   ```

   Used by Mox's public listener for SMTP STARTTLS, SMTPS (465), IMAPS (993).

2. **Traefik TLS** — `/opt/pangolin/config/traefik/certs/`

   ```text
   fullstacked-mail-chain.pem  (root:root 0644)
   fullstacked-mail-key.pem    (root:root 0640)
   ```

   Loaded by Traefik via file provider in `/opt/pangolin/config/traefik/dynamic_config.yml`:

   ```yaml
   tls:
     certificates:
       - certFile: /etc/traefik/certs/fullstacked-mail-chain.pem
         keyFile: /etc/traefik/certs/fullstacked-mail-key.pem
   ```

   Note: The Traefik container maps `/opt/pangolin/config/traefik/certs/` to `/etc/traefik/certs/`.

After copying, the service:
- Validates both cert/key pairs with `openssl x509 -noout` and `openssl pkey -noout`
- Touches the dynamic config file to trigger Traefik reload
- Restarts Mox if it's already running

## Mox TLS Config

In `/var/lib/mox/config/mox.conf`, public listener:

```sconf
TLS:
	KeyCerts:
		-
			CertFile: tls/fullstacked-mail-chain.pem
			KeyFile: tls/fullstacked-mail-key.pem
```

One SAN cert covers all three hostnames (mail.fullstacked.se, mta-sts.fullstacked.se, autoconfig.fullstacked.se).

## DNS Prerequisites

Before certificate issuance, Cloudflare must have DNS records (DNS-only, not proxied):

```text
mail.fullstacked.se A 143.14.50.130
mta-sts.fullstacked.se CNAME mail.fullstacked.se
autoconfig.fullstacked.se CNAME mail.fullstacked.se
```

## Validation

```sh
sudo -u mox sh -c 'cd /var/lib/mox && mox -config config/mox.conf config test'
systemctl status mox-lego-cert.timer --no-pager
systemctl status mox --no-pager

openssl s_client -connect mail.fullstacked.se:25 -starttls smtp </dev/null 2>&1 | grep 'subject\|Verify'
openssl s_client -connect mail.fullstacked.se:993 </dev/null 2>&1 | grep 'subject\|Verify'
```

## Special Constraint

Lego state (`/var/lib/mox-lego/`) must be preserved across NixOS rebuilds. Do not move it into the Nix store. The `mox-lego-cert` service uses `ReadWritePaths` to allow access to this directory from within the systemd sandbox.
